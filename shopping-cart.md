# 智慧無障礙自動結帳系統 - 開發對話紀錄

## 📅 紀錄資訊

* **專案類型**：前端網頁設計 (HTML/CSS/JS)
* **核心需求**：自動結帳、無障礙、RWD、酒精辨識機制、貨架挑選系統、隨機換貨功能。

---

## 🕒 對話時間線

### 第一階段：概念設計與需求分析

* **我**：提出設計一款「全自動購物車結帳系統」的需求。要求包含無障礙設計、RWD、人經過自動顯示清單、錯誤報錯按鈕，以及針對酒精飲料的店員身分確認機制。
* **Gemini**：作為前端工程師，提出了介面流程設計、A11y 無障礙標準（WCAG 2.1 AA）、以及使用 WebSocket/API 進行酒精驗證的技術架構。

### 第二階段：原型開發 (靜態背景版)

* **我**：要求提供完整的 HTML+CSS+JS，並設定背景為放滿蔬菜水果的貨架。
* **Gemini**：交付了第一個版本。
* **技術特點**：使用 `setTimeout` 模擬人經過感應器的自動觸發效果。
* **視覺設計**：CSS 模擬超市層架背景，並使用大字體確保無障礙讀取。



### 第三階段：互動升級 (玩家挑選模式)

* **我**：增加互動需求。玩家必須能在貨架上點選 5 種不同品項，確認數量後才能進入結帳窗口，且購物車需有「清空」選項。
* **Gemini**：將自動觸發改為「互動選購」。
* **功能更新**：新增商品點擊 Badge 數字標籤、清空購物車邏輯（`confirm` 視窗）、以及整合酒精檢核邏輯。



### 第四階段：代碼重構與規格整合

* **我**：要求將 `getElementById` 全部替換為 `querySelector`，並整合所有程式碼。
* **Gemini**：重構程式碼，確保語法現代化。
* **整合點**：將 HTML/CSS/JS 合併為單一檔案，並優化 CSS Grid 佈局以符合更高標準的 RWD。



### 第五階段：功能強化 (隨機性與支付回饋)

* **我**：新增「隨機更新貨架」按鈕，讓商品能換批出現；並要求點下支付按鈕後顯示「結帳完成」的結果。
* **Gemini**：引入商品資料池（Pool）。
* **新增邏輯**：使用洗牌演算法（Shuffle）實作隨機商品換貨；實作支付成功後的動態 UI 切換（Success Screen）。



### 第六階段：最終整合 (刪除功能與視覺裝飾)

* **我**：要求在購物車新增「刪除單一品項」按鈕，並在背景加上「SALE」圖案與貨架視覺。
* **Gemini**：交付最終完整版本。
* **視覺強化**：加入 3D 立體木質貨架 CSS 與 `SALE` 水印背景。
* **操作優化**：結帳清單中每個 Item 都有獨立的刪除按鈕，並會動態連動酒精驗證狀態。

## 🛠️ 技術規格摘要

| 分類 | 內容 |
| --- | --- |
| **開發語言** | HTML5, CSS3, JavaScript (Vanilla JS) |
| **選擇器規格** | 嚴格執行 `querySelector` / `querySelectorAll` |
| **無障礙設計** | ARIA-live 狀態更新、高對比色、大尺寸觸控區域 |
| **關鍵算法** | 動態商品池隨機抽樣、購物車狀態同步、身分核對非同步模擬 |
| **視覺特效** | CSS Linear-gradients 擬真貨架、動畫 UI 轉換、水印背景 |
